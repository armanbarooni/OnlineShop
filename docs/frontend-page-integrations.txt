General observations
=====================================================================
1. Only five services exist under front/FA/public/assets/js/services (cart, checkout, comparison, payment, product). All other services referenced by HTML files (wishlist, review, coupon, order, return, auth, api-client, utils, auth-guard, chart dependencies) are missing. Identical filenames live in presentation/assets/js and appear to be the intended implementations; copying or bundling them into front is required for functionality.
2. Multiple inline handlers reference undefined functions: handleVariantSelection (product page), toggleOffcanvas/closeOffcanvas/toggleDropdown (navigation across pages), sendOTP/togglePassword/switchTab (product modal & register page) because shared scripts are absent.
3. Many actions only manipulate UI state or localStorage and never call backend even when services exist (e.g., addToWishlist in last-viewed page). After restoring services, review logic to ensure endpoints align with backend routes (/api/Wishlist, /productreview, /api/orders, etc.).
4. Navigation paths are inconsistent (`viewProduct` on last-viewed sends `/product/${id}` while actual file is product.html?id=...).
5. To unblock functionality: move/copy presentation/assets/js/{api-client.js,auth-service.js,auth-guard.js,utils.js,services/*.js,dependencies/app.js,plugin/chart-js/chart.js,...} into front/FA/public/assets/js (respecting subfolders) or adjust build tooling to serve shared bundle.
